# Redundant Layer 2/Layer 3 Campus Network Lab

Cisco Packet Tracer lab implementing a dual-core redundant switching design with RSTP, LACP EtherChannel (L2 + L3), VLAN segmentation, inter-VLAN routing, and port security.

## Topology

- **R1** — edge router, dual-homed to both multilayer switches
- **MLS1 / MLS2** — multilayer switch core, cross-connected to both access switches for full redundancy
- **SW1 / SW2** — access layer switches, one per building/floor
- **VLANs:** 10 (Native), 20 (Admin), 30 (IT), 40 (HR), 50 (Sales)
- **Redundancy:** RSTP with split primary/secondary root per VLAN group across MLS1/MLS2; LACP EtherChannel on every inter-switch link; dual static routes from R1 to every VLAN subnet

| Device | Interface(s) | Connects to | Channel |
|---|---|---|---|
| MLS1 | Gi1/0/1 | R1 | routed, no channel |
| MLS1 | Gi1/0/2–3 | MLS2 | Po2 |
| MLS1 | Gi1/0/4–5 | SW1 | Po3 |
| MLS1 | Gi1/0/6–7 | SW2 | Po5 |
| SW1 | Gi0/1–2 | MLS1 | Po3 |
| SW1 | Fa0/4–5 | MLS2 (backup) | Po4 |
| SW1 | Fa0/1–3 | PC1/PC2/PC3 | access |
| R1 | Gi0/0 | MLS1 | 192.168.101.0/30 |
| R1 | Gi0/1 | MLS2 | 192.168.102.0/30 |

> MLS2/SW2 interface numbering mirrors MLS1/SW1 by design (Po1 = MLS2↔SW2, Po4 = MLS2↔SW1) — confirm against your own `show run` if replicating this lab, since those two devices' configs weren't captured here.

## Troubleshooting Commands Used, in Diagnostic Order

**1. Physical/interface layer**
```
show interfaces status
show interfaces <int>
show interfaces <int> counters errors
```

**2. VLAN/trunking**
```
show vlan brief
show interfaces trunk
show interfaces <int> switchport
```

**3. Spanning tree**
```
show spanning-tree summary
show spanning-tree
show spanning-tree vlan <id>
show spanning-tree interface <int> detail
```

**4. Err-disabled port recovery (BPDU Guard trip)**
```
interface <int>
 shutdown
 no shutdown
```
or auto-recover:
```
errdisable recovery cause bpduguard
errdisable recovery interval 30
```

**5. EtherChannel/LACP**
```
show etherchannel summary
show etherchannel load-balance
show lacp neighbor
```

**6. Layer 3/routing**
```
show ip route
show ip interface brief
ping <destination>
tracert <destination>
```

## Known Packet Tracer Limitation

`show spanning-tree` reports point-to-point EtherChannel/routed links (Po2, Po3, Po5) as link type **Shr (Shared)** instead of **P2P**. Packet Tracer's multilayer switch ports don't correctly negotiate full-duplex on these simulated links — this is a simulator limitation, not a misconfiguration, and RSTP still converges correctly.
